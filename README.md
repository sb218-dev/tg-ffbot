# 🍔 TG FastFood Backend

Серверная часть для Telegram Mini App по заказу еды. Реализует API для клиентского приложения, админ-панель для кухни и управление меню через бота/API.

## 📂 Структура

Основной файл логики: `server.js`.
Статика (клиентская часть) должна располагаться в папке `public/`.

##  Функционал

### 🤖 Telegram Bot
- **Команда `/start`**: Отправляет кнопку для открытия Web App.
- **Уведомления**:
  - Отправка новых заказов в чат кухни (`KITCHEN_CHAT_ID`).
  - Уведомление пользователя о готовности или отмене заказа.
  - Автоматическое уведомление о задержке заказа (если текущее время > времени готовности).

### 🛒 API Клиента
- `GET /api/locations`: Список доступных заведений.
- `GET /api/menu`: Получение меню для выбранной точки (с учетом стоп-листов).
- `POST /api/order`: Оформление предзаказа (проверка времени работы, валидация позиций).
- `GET /api/my_orders`: Просмотр активных заказов пользователя.
- `POST /api/orders/:id/cancel_by_user`: Отмена заказа пользователем.

### 👨‍🍳 API Админки и Кухни
**Защита**: Basic Auth (User: `PIDIDI`, Pass: `EPSTEIN`).

- `GET /kitchen.html`: Интерфейс кухни (защищен паролем).
- `GET /api/orders`: Список всех заказов для кухни.
- `POST /api/orders/:id/status`: Изменение статуса (ready/cancelled).
- **Управление меню**:
  - CRUD операции для локаций и позиций меню.
  - Управление добавками (addons) и связями товаров.
  - Стоп-лист (toggle availability).

## 💾 База данных (SQLite)

При первом запуске автоматически создается файл `database.sqlite` и таблицы:
1. **locations**: Точки продаж (время работы, активность).
2. **menu**: Блюда, категории, цены, тип (main, addon, both).
3. **item_addons**: Связи блюд с добавками/соусами.
4. **orders**: История заказов, статусы, тайминги.

*При пустой базе происходит автоматическое наполнение тестовыми данными (сидирование).*

## 🛠 Установка и запуск

### 1. Установка зависимостей

```bash
npm install express node-telegram-bot-api sqlite3 cors express-basic-auth
```

### 2. Установка зависимостей

```bash
npm install
```

### 3. Настройка окружения

Создайте файл `.env` в корне проекта и добавьте следующие переменные:

```env
BOT_TOKEN=ваш_токен_от_BotFather
DB_URI=ссылка_на_базу_данных
ADMIN_ID=id_администратора
```

### 4. Запуск

Режим разработки:
```bash
npm run dev
```

Продакшн режим:
```bash
npm start
```

## 🤖 Команды бота

- `/menu` — Открыть меню
- `/cart` — Посмотреть корзину
