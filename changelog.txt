======================================================
 ИСТОРИЯ ИЗМЕНЕНИЙ ПРОЕКТА: TG FASTFOOD MVP
======================================================

1. АРХИТЕКТУРА И РЕФАКТОРИНГ
------------------------------------------------------
- Монолитный файл server.js разделен на логические модули для удобства поддержки.
- Создан файл database.js, отвечающий за инициализацию SQLite, настройку WAL-режима и первичное наполнение (сидирование) таблиц.
- API разбито на роутеры:
  - routes/admin.js: логика управления меню, локациями и заказами для кухни.
  - routes/client.js: логика витрины, корзины и профилей для клиентов.
- Фронтенд клиента разделен на 3 файла: index.html (каркас), app.js (логика) и style.css (дизайн).

2. КЛИЕНТСКОЕ ПРИЛОЖЕНИЕ (WEB APP)
------------------------------------------------------
- Интеграция Яндекс Карт: Добавлена интерактивная карта для выбора точек. Реализована ленивая загрузка (lazy loading) скрипта карт для ускорения старта приложения.
- Дизайн и брендинг: Внедрена фирменная палитра (синий #3F5CA9, оранжевый #F48C5B, фон #E9EEF5). Добавлена шапка с логотипом.
- Каталог (Сетка): Меню переведено в формат плиток (3 позиции в ряд). Категории вынесены в горизонтальное липкое меню (sticky) с плавным авто-скроллом к нужному разделу.
- Изображения: Внедрена поддержка загрузки фото в современном формате .webp из локальной папки /public/images/. Картинки привязываются по названию блюда (чтобы не дублировать их для разных точек). Добавлен fall-back генератор красивых заглушек при отсутствии фото.
- Пользовательские профили: Добавлена прозрачная регистрация по Telegram ID. Приложение запоминает последнюю выбранную точку и при следующем входе сразу открывает её меню.
- Поиск: Добавлена функция поиска блюд. Для оптимизации работы на слабых телефонах внедрен механизм Debounce (задержка фильтрации в 200мс при быстром вводе текста).
- UI/UX: Добавлены плавные анимации перехода между экранами (cubic-bezier). Внедрены визуальные эффекты упругого нажатия на кнопки (:active) и тактильный отклик (Haptic Feedback) устройства.
- Корзина: Перенесена на отдельный экран. Главная кнопка Telegram (MainButton) теперь умно меняет состояния и становится серой/некликабельной при пустой корзине.

3. ПАНЕЛЬ УПРАВЛЕНИЯ И КУХНЯ (KITCHEN.HTML)
------------------------------------------------------
- Перенос времени: В карточку заказа добавлена функция редактирования (переноса) времени готовности заказа поваром.
- Авто-уведомления: При переносе времени сервером автоматически отправляется сообщение в Telegram клиенту с новым временем.

4. БЭКЕНД, ОПТИМИЗАЦИЯ И ПРОИЗВОДИТЕЛЬНОСТЬ
------------------------------------------------------
- Сжатие трафика: Добавлен пакет `compression`. Теперь все ответы сервера и JSON сжимаются через GZIP, что критически ускоряет загрузку при слабом мобильном интернете (3G).
- Кэширование: Для статических файлов (особенно фотографий) настроено жесткое кэширование браузером на 7 дней (заголовки Cache-Control).
- Оптимизация SQLite: 
  - Включен режим WAL (Write-Ahead Logging), позволяющий читать и писать в базу одновременно, избегая очередей.
  - Добавлены индексы для колонок, по которым происходит частый поиск (tg_id, status, location_id).
- Smart-Архивация (Cron Job): Настроен умный фоновый процесс. Ровно в 00:00 (по МСК) сервер собирает все вчерашние заказы, сжимает их в .gz архив, раскладывает по папкам (Год-Месяц-День/Точка) и удаляет из базы, после чего сжимает сам файл базы данных командой VACUUM.

5. ДОКУМЕНТАЦИЯ
------------------------------------------------------
- Полностью обновлен файл README.md. Указана актуальная архитектура, стек технологий, список добавленных фич (от стоп-листов до смарт-архивации) и правильные шаги для запуска.

6. БЕЗОПАСНОСТЬ И КОНФИГУРАЦИЯ
------------------------------------------------------
- Секретные данные (TOKEN, KITCHEN_CHAT_ID, WEBAPP_URL) вынесены из основного кода в отдельный файл config.js.
- Создан файл .gitignore для предотвращения случайной утечки базы данных (database.sqlite), архивов (archive/) и секретных ключей (config.js) в публичный репозиторий.
